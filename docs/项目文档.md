# 应急转贷业务管理系统项目文档

## 1. 项目概述

本项目是“应急转贷业务管理系统”的前端静态原型，主要用于展示业务办理、审批、放款和报表上报相关页面。当前版本可以直接在浏览器中运行，适合用于需求沟通、页面验收、流程演示和后续开发参考。

项目恢复位置：

```text
D:\code\yjzd
```

源压缩包：

```text
D:\桌面\yjzd.zip
```

## 2. 技术栈

- HTML：页面结构与静态业务数据。
- CSS：后台管理系统样式、卡片、表格、详情页、弹窗和响应式布局。
- JavaScript：页签切换、详情页打开/返回、弹窗显示/关闭、分页、按钮交互。
- 无构建工具、无后端依赖、无第三方前端框架。

## 3. 业务模块

### 首页

首页包含消息中心、运行指标和累计服务成效。消息中心展示业务审批、放款审批、报表审批的待办数量，并可跳转到对应模块。

### 业务全流程

展示应急转贷业务项目的流程节点，包括授信、合同放款等阶段，同时展示客户名称、客户号、负责部门、授信开始时间和主办业务经理。

### 业务审批

用于展示待处理的授信审批和业务变更审批。支持进入详情页查看授信信息、客户信息、风控报告、审批意见，并提供同意、退回、拒绝等操作弹窗。

### 放款审批

展示放款申请列表和详情，包括合同号、放款金额、合同起始日、合同到期日、担保信息、放款审批意见等内容。

### 报表上报审批

展示月度经营指标完成情况表，包括报表期、报表名称、主要经营指标、审批意见和历史记录。

## 6. 当前限制

- 这是静态原型，不包含真实登录、用户体系、权限控制、接口请求或数据库。
- 表单提交、审批确认、撤回等操作只是前端演示，不会保存到服务器。
- 搜索、重置和部分按钮目前主要用于演示交互，不具备完整业务查询能力。
- 页面中存在样例手机号、身份证号、企业信用代码等字段，正式交付或演示前建议脱敏处理。

## 7. 后续开发建议

1. 梳理真实业务流程，确定授信、放款、变更、报表审批的状态流转。
2. 定义后端接口，包括列表查询、详情查询、审批提交、附件上传、审批记录查询。
3. 将静态 HTML 拆分为组件化页面，例如 Vue、React 或现有团队常用前端框架。
4. 接入登录、组织、角色、权限和审批人配置。
5. 建立数据字典，包括业务类型、区县、客户类型、审批状态、报表期等。
6. 对演示数据做脱敏，并区分 mock 数据和真实业务数据。
7. 增加表单校验、错误提示、加载态、空状态和操作日志。
